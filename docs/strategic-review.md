# Strategic Review

## Has Product Finder Evolved Beyond Being A Bargain Finder?

Yes.

Product Finder still finds bargains, but that is now one output of a broader platform.

It is becoming a market knowledge and buying-intelligence platform for second-hand and mixed-market acquisition.

## What It Is Becoming

Product Finder is becoming a system that builds and applies knowledge about:

- which sources can be used safely
- how reliable those sources are
- what listings exist right now
- what products those listings describe
- whether two listings are the same opportunity
- what products cost new and used
- whether a listing's price is committed, ambiguous, suspicious, or actionable
- what a user's project is actually trying to buy
- whether a decision should be buy, watch, ignore, review, or wait

The bargain is the final presentation. The architecture is the knowledge pipeline beneath it.

## How This Changes Architectural Thinking

### From Source Integration To Connector Platform

The strategic question is no longer "can we add a marketplace?"

It is:

- can this connector operate legitimately?
- what knowledge does it contribute?
- how fresh is it?
- how trustworthy is it?
- how does it enrich identity, product, price, or listing understanding?

### From Static Prices To Market Evidence

Item normal price is now a fallback, not the destination.

The strategic goal is evidence-backed pricing:

- used observations
- auction close data
- retailer price candidates
- trend confidence
- source-weighted evidence

### From Matching Keywords To Understanding Products

Search terms are still useful, but the durable asset is product understanding.

The catalogue should grow into a model of products, variants, accessories, compatibility, alternatives, and bundles.

### From Score To Decision

Deal score is useful, but it is not a full recommendation.

A recommendation needs:

- product identity
- price confidence
- listing trust
- source coverage
- duplicate state
- project priority
- user preference

### From Local Tool To Platform With Ownership

The current local app is still the right runtime. But the domain model now clearly supports future public and signed-in layers.

The architecture should keep public discovery and signed-in ownership separate.

## Roadmap Implications

Near-term work should favour:

- connector coverage and quality
- catalogue quality
- listing understanding
- identity resolution
- market evidence
- trust foundations
- ownership-safe public readiness

It should avoid:

- cosmetic product expansion ahead of evidence quality
- commercial mechanics ahead of ownership
- AI recommendations ahead of trustworthy knowledge layers
- more source integrations that do not declare compliance/risk/quality

## Strategic Principles That Emerged From Implementation

- The platform's memory is more valuable than any single search.
- Product identity is the core bottleneck behind scoring quality.
- Source compliance has to be modelled, not merely stated.
- A human review queue is a reusable architecture pattern.
- Objective deal quality and personal importance are different signals.
- A listing's raw price is not always an actionable price.
- Marketplaces should be replaceable; connector declarations should not be.
- Public/commercial features must consume the platform model, not reshape it.

## Strategic Recommendation

Treat Product Finder's next stage as platform hardening and knowledge expansion, not feature accumulation.

The highest-leverage question before implementation should be:

Which knowledge layer does this strengthen, and what evidence will prove it worked?
