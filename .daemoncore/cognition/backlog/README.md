# Backlog Workspace (Canonical Planning)

This directory is the canonical planning source of truth for this repository.

- Epics, features, user stories, and tasks that are **ready for execution** live here.
- Ideas and experiments should start life in `cognition/ideas/` and be promoted once validated.

The exact backlog schema is up to the project, but a common pattern is:

- A `backlog.json` rollup file.
- Per-entity JSON files in subfolders (e.g. `epics/`, `features/`, `stories/`, `tasks/`).

Initially this folder may be empty; it exists so agents know where to place structured work items once you adopt a backlog.
